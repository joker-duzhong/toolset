# 开发任务：【语筑 NestTalk】房产 AI 助手核心模块开发

## 0. 接口规范与前提约束（🔴 核心指令）
**致 AI 开发助手**：在开始编写任何代码之前，你必须先访问并读取完整的 OpenAPI 文档：`https://api.lxyy.fun/docs`。
- **重点关注**：`Tags` 为「语筑」下的所有接口。
- **强制要求**：绝对不要自行编造 API 路径和请求/返回结构！所有接口的入参（如 `session_id` 的传递）、出参和字段名，必须严格遵循文档中的定义。用户登录鉴权系统已完成，直接复用现有的 Token/Auth 逻辑即可。

## 1. 业务与模块设计
本期需求为移动端优先（适配微信环境）的 H5/Web 应用，包含三大核心模块：

1. **首页监控大盘 (Dashboard)**：管理用户的买房监控任务。
2. **多轮需求对话 (Chat-to-Task)**：通过自然语言聊天，逐步明确需求并最终生成监控任务。
3. **捡漏雷达 (Property Radar)**：展示被监控任务捕捉到的具体房源及 AI 评估。

## 2. 详细任务清单 (TODO)

### 阶段一：首页监控大盘 (Home Dashboard)
- [ ] **空状态 (Empty State) 设计**：
    - 当用户没有监控任务时，展示友好的空状态提示（如“暂无房产监控”）。
    - 放置一个醒目的 CTA 按钮：“唤醒语筑，创建监控”，点击跳转至多轮对话页。
- [ ] **监控任务卡片列表**：
    - 展示已创建的监控条件摘要（如：高新区·套三·200万内）。
    - **卡片功能 1**：启用/禁用开关 (Toggle)，用于控制该条件下的监控状态。
    - **卡片功能 2**：展示“微信推送已开启/未开启”状态，若未绑定，提供引导绑定的交互入口。
- [ ] **动画与交互**：使用 `framer-motion` 为列表加载和卡片展开/收起添加流畅顺滑的过渡动画。

### 阶段二：AI 多轮对话与任务创建 (Chat Flow)
- [ ] **聊天界面开发**：
    - 类似常见的对话流界面（气泡式）。底部为输入框。
    - 预设引导语（如：“请告诉我你想在哪个区域买房？预算多少？”）。
- [ ] **上下文记忆 (Session 管理)**：
    - 首次发送消息时，解析后端返回的 `session_id` 并保存在本地状态。
    - 后续的所有多轮对话请求，必须在 Payload 中带上该 `session_id`，以保持上下文记忆。
- [ ] **任务确认与创建**：
    - 当 AI 判断信息收集完整后，通过特定接口或标识返回结构化数据预览。
    - 前端展示“需求确认卡片”，用户点击“确认并开启监控”后，调用对应的 Task 创建接口，随后跳转回首页或雷达页。

### 阶段三：捡漏房源雷达 (Property Radar)
- [ ] **房源信息流卡片**：
    - 接收并展示匹配的房源列表。
    - 核心字段：小区名、总价、单价、户型、面积。
    - **特殊视觉**：对“降价”、“低均价”等高价值信息做高亮/角标处理（如“低于同小区均价15%”）。
- [ ] **AI 价值评估面板**：
    - 展示后端返回的 AI 深度分析数据（如：优缺点总结、历史成交对比、走势研判）。

## 3. 设计规范与技术约束
- **UI 风格**：「语筑」与「灵钥工坊」品牌基调。极简、克制、高效（类似 Vercel 或 Linear 的设计语言）。绝对不要堆砌花哨无用的渐变和阴影。
- **移动端优先**：确保在微信内置浏览器下的 100vh 表现正常，底部输入框不会被系统键盘遮挡，滑动体验如丝般顺滑。
- **动画库**：强制使用 `framer-motion`。需要包含：页面切换动画、卡片弹跳出现、对话气泡浮现动画。

---

## 4. 验收标准 (Definition of Done)
1. **接口真实连通**：AI 对话时的 `session_id` 必须在 Network 中清晰可见且正确传递，断点刷新不能乱序。
2. **闭环体验**：新用户进入 -> 看到空状态 -> 点击对话 -> 经过 2-3 轮对话后确认 -> 首页出现监控卡片 -> 雷达页展示相关房源。
3. **视觉顺滑**：使用 Framer Motion 实现的动画在移动端不卡顿，没有突兀的 DOM 闪烁。
4. **代码整洁**：组件拆分合理（如拆分 `TaskCard`, `ChatMessage`, `PropertyCard`），无冗余内联样式。
